# AI実装のための要件整理ツール

機能のアイデアはあるのに、  
「画面のあとに何が起きるのか」  
「どこに保存するのか」  
「どこから読み出すのか」  
あたりで急に難しくなることがあります。

このツールは、そうした内容を**箱と線で見える化**しながら整理し、  
**AIやエンジニアに渡しやすい形**にまとめるための、**ローカル動作の単一HTMLツール**です。

非エンジニアでも、頭の中にあるアイデアを  
**画面 / 入力 / 保存 / 参照 / 出力** のつながりとして整理し、  
段階的なチェックを通して、説明用・相談用・実装用の情報へ落とし込めます。

アプリ本体は index.html 1ファイルで完結しています。  
外部ライブラリ、フレームワーク、CDN、バックエンドは使っていません。

---

## このツールでできること

- 箱を追加 / 編集 / 削除 / 移動できる
- 図エリアをドラッグして箱を作れる
- 箱の大きさを変えられる
- 親箱を追加 / 編集 / 削除できる
- 箱の中に細かい項目を追加 / 編集 / 削除できる
- 箱同士を線でつなげられる
- 箱同士をドラッグで接続できる
- 箱の中の項目同士もつなげられる
- Undo / Redo が使える
- Ctrl + ホイール / Ctrl + ↑↓ でズームできる
- 3段階のチェックで不足やズレを確認できる
- チェック内容を日本語で表示できる
- JSONで保存 / 読込できる
- 古い形式のJSONもできる範囲で安全に読める
- AI向けの整理結果を出力できる
- 人に説明しやすい文章として出力できる
- AIに相談するためのJSONを出力できる
- 初期サンプルからすぐ試せる

---

## こんなときに向いています

- 作りたい機能のイメージはあるが、バックエンドやDBの話になると難しく感じる
- 画面の話はできるが、保存や参照のつながりが整理できない
- AIに相談したいが、何をどう渡せばよいか分からない
- 顧客向けの説明と、実装側の整理を両立したい
- ツールをローカルだけで動かしたい

---

## 起動方法

1. index.html をブラウザで直接開きます。
2. 左カラムで箱や親箱を追加します。
3. 中央の図エリアで箱を動かし、線をつなぎます。
4. 右カラムで箱 / 線 / 箱内項目の詳細を編集します。
5. STEP1、STEP2、STEP3 を順に実行して、足りない点やつながりのズレを確認します。
6. 必要に応じて JSON保存、JSON読込、AI向け出力、説明用出力を使います。
7. AIに相談したいときは AI相談JSON出力 を使います。

---

## 使い方のイメージ

このツールでは、アイデアをいきなりコードにするのではなく、まず**つながる形に整理**します。

たとえば、こんな流れです。

1. 「こんな画面がほしい」「こういう操作をしたい」というアイデアを箱で置く
2. 箱同士を線でつないで、流れや関係を見える化する
3. 「入力した内容はどこに保存されるか」「画面は何を読み出して表示するか」を埋める
4. 3段階チェックで、抜けやあいまいさを見つける
5. まとまったら、人向けの説明やAI向けの出力に変換する

## 画面の見方

### 左カラム
- 基本設定  
  プロジェクト名、現在のSTEP
- 操作  
  箱追加、親箱追加、線接続開始 / 箱接続に戻る、削除、サンプルへ戻す、Undo / Redo
- 保存と読込  
  JSON保存、JSON読込、AI向け出力、説明用出力、AI相談JSON出力
- 使い方  
  操作の短い手順、STEP説明

### 中央カラム
- 図エリア  
  箱、親箱、箱内項目、線の表示 / ドラッグ箱作成 / 箱リサイズ / ドラッグ線接続
- 接続ガイド  
  線接続中の案内
- 状態バッジ  
  現在STEP、プロジェクト名、箱数 / 線数、表示倍率

### 右カラム
- 詳細編集  
  箱編集、線編集、箱内項目編集
- 現在状態の要約  
  件数、カテゴリ内訳、接続モード、表示倍率、STEP状況
- STEP結果表示  
  STEP1 / STEP2 / STEP3 の結果を個別表示

## このツールで整理するもの

### 箱
箱は、アイデアのかたまりです。

- 通常の箱  
  やりたいこと、画面、入力、保存先、参照先、出力、補足メモなどを表します。
- 親箱  
  複数の箱をまとめる大きな枠です。
- 箱内項目  
  箱の中に入れられる細かいメモや要素です。

### 線
線は、箱や項目どうしのつながりです。

- 箱同士の線  
  全体の流れや関係を表します。
- 箱内項目同士の線  
  より細かいレベルのつながりを表します。
- 関係種別  
  表示、入力、保存、参照、出力、関連

---

## 3段階チェックの役割

このツールでは、図を描くだけで終わらず、3段階で確認できます。

### STEP1: 大きな抜け漏れチェック
主な箱が足りているか、線がつながっているかを見ます。

- 主要カテゴリ不足
- カテゴリ未設定
- タイトル未入力
- 独立した箱
- 線がない状態

### STEP2: 保存と参照のつながりチェック
「どこに保存するか」「どこから読むか」があいまいでないかを見ます。

- 保存データと保存線の対応
- 参照データと参照線の対応
- 保存元不明
- 参照先不明
- 画面表示の参照不足
- ユーザー入力の行き先不足

### STEP3: 人に説明できるかチェック
後から見返しても分かるか、顧客にも説明できるかを見ます。

- 顧客向け注釈不足
- 線説明不足
- 意味合い説明不足
- 顧客要望の説明不足
- 後で見返したときに分かりにくい要素

## 保存と読込

### JSON保存
途中保存 / 再開用のデータとして保存できます。

保存対象:
- version
- projectName
- currentStep
- nodes
- edges

### JSON読込
保存したJSONを読み込んで、続きから再開できます。

読込時の考え方:
- 現行JSONを優先して読み込みます
- 古い形式の boxes / lines / connections も、読める範囲で吸収します
- 不正な参照先を持つ線や、存在しない親箱参照は安全側に補正します

> AI相談JSON出力 は、AIに相談するための別出力です。  
> 途中保存 / 再開用ではないため、JSON読込 の対象ではありません。

詳細は docs/JSON_SCHEMA.md を参照してください。

---

## 出力の種類

### AI向け出力
AIやエンジニアが読み取りやすいように、構造を整理して出力します。

- IDを含む構造化テキスト
- 箱、親箱、箱内項目、線、方向、STEP状況
- 顧客要望と関連実装要素の対応

### 説明用出力
人に説明しやすい自然文として出力します。

- 図全体の目的
- 各箱の意味
- 各線の意味
- 入力 → 表示 → 保存 → 参照 → 出力 の流れ
- 顧客向け注釈

### AI相談JSON出力
AIに「何が足りないか」「何を確認すべきか」を相談しやすい形で出力します。

- ファイル名は <projectFileBase()>_AI相談.json
- format version projectName currentStep consultationPurpose stepGuide を含みます
- graphSummary validation activeConcerns graph aiRequest を含みます
- 保存 / 参照 / 表示 / 入力のあいまいさをAIに相談しやすい構造です
- 保存用JSONとは分離しており、途中保存 / 再開には使いません

## 最初の土台づくりにAIを使う流れ

このツールは、最初の案がまだあいまいな段階でも使えます。

1. まだ案がざっくりしていても、要望メモと index.html をAIに渡します。
2. index.html 内コメントにある「AIへ渡す定型文」を使い、保存用JSON互換を JSON のみで返してもらいます。
3. 返ってきたJSONを JSON読込 で読み込んで、図の土台を出します。
4. その後に箱 / 親箱 / 箱内項目 / 線を足して育てます。
5. 詰まったら AI相談JSON出力 をAIに渡して、不足点や確認点を整理します。

## ファイル構成

### ルート
- index.html  
  アプリ本体です。HTML / CSS / JavaScript を1ファイルにまとめています。
- README.md  
  このプロジェクトの概要、使い方、ファイル説明です。
- AGENTS.md  
  このリポジトリで作業するときの厳守ルールです。
- REQUIREMENTS.md  
  ユーザー要望の原文と要望メモです。承認前の相談や整理の起点として使います。

### docs
- docs/SPEC.md  
  ユーザー承認後に固定した仕様書。既存仕様の変更は元記述の下へ日付付きで追記して管理します。
- docs/IMPLEMENTATION_RULES.md  
  実装時の共通ルール
- docs/COMPLETED_REQUIREMENTS.md  
  達成済みの要件一覧
- docs/UNMET_REQUIREMENTS.md  
  承認済みかつ未実装の要件一覧
- docs/VALIDATION_RULES.md  
  STEP1 / STEP2 / STEP3 の判定条件と文言
- docs/JSON_SCHEMA.md  
  保存 / 読込する JSON の現行仕様

手動確認が必要な項目は、固定 Markdown ではなくチャット返信で案内します。

## 追記: 最近の操作改善

- 選択中の箱 / 線は Delete キーでも削除できます。右カラムの入力欄や textarea に文字を入れている間は、図の削除は起きません。
- 親箱を追加 は、次の空き領域ドラッグで親箱を作るモードになりました。
- 通常箱を親箱の中へ移動して離すと、その親箱の子要素として parentId が付きます。親箱の外へ十分に出して離すと解除されます。
- JSON読込 は、途中保存した保存用JSONだけでなく、AIがこのHTML向けに作った保存用JSON互換のひな型も読めます。
- AI相談JSON出力 は別用途です。AIに不足点や確認点を相談するための出力であり、JSON読込 の対象ではありません。
