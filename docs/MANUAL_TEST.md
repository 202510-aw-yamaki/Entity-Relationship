# 手動確認手順

## 前提
- `index.html` をブラウザで直接開く
- 初期表示でサンプルデータが見えることを確認する

## 画面表示
- 左 / 中央 / 右の3カラムが崩れていない
- 左カラムに操作、保存 / 読込、使い方が見える
- 中央に図エリア、右カラムに詳細編集とSTEP結果が見える
- 初期サンプルで箱、親箱、線、箱内要件が表示される

## 箱と親箱
- `箱を追加` で通常の箱を追加できる
- `親箱を追加` で親箱を追加できる
- 追加した箱を選ぶと右カラムでカテゴリ、タイトル、意味合い、顧客向け注釈を編集できる
- 通常の箱を親箱に所属させられる
- 親箱を通常の箱に戻したとき、子箱の所属が不正にならない
- 箱と親箱を削除できる

## 箱内要件
- 通常の箱を選び、`要件を追加` で箱内要件を追加できる
- 箱内要件の名前と説明を編集できる
- 箱内要件を削除すると、対応する線も消える

## 線
- `線接続を開始` 後に箱同士を選ぶと箱同士の線を作れる
- `線接続を開始` 後に箱内要件同士を選ぶと箱内要件同士の線を作れる
- 作成した線を選ぶと関係種別、説明、矢印を編集できる
- 線を削除できる
- 同じ向きの重複線が作られない

## 移動
- 通常の箱をドラッグで動かせる
- 親箱をドラッグで動かせる
- 移動後も線表示が追従する

## JSON保存 / 読込
- `JSON保存` で現在状態を保存できる
- 保存したJSONを `JSON読込` で読み戻せる
- 読込後に箱、親箱、箱内要件、線、STEP、プロジェクト名が復元される
- 読込後に再編集、再保存、再判定ができる
- 旧JSONを読み込んでも落ちず、読める範囲で復元される

## STEP判定
- STEP1判定で不足や独立箱が日本語文で出る
- STEP2判定で保存 / 参照の不足が日本語文で出る
- STEP3判定で説明不足が日本語文で出る
- 問題がない場合は問題なしと分かる文が出る
- 図を編集した後、再判定が必要な状態が分かる

## 出力
- `AI向け出力` を実行できる
- `human向け出力` を実行できる
- AI向け出力にID、箱、線、関係種別、方向、STEP情報が入る
- human向け出力に図全体の目的、各箱の意味、各線の意味、流れ、顧客向け注釈が入る

## サンプル確認
- 初期サンプルに親箱がある
- 初期サンプルに顧客要望、画面表示、ユーザー入力、保存データ、参照データ、出力結果が含まれる
- 初期サンプルに箱内要件と箱内要件同士の線が含まれる

## 確認後の見方
- 問題があれば、どの操作で何が起きたかを記録する
- JSON読込の不具合は、使ったJSONの版と不足した項目を合わせて記録する
