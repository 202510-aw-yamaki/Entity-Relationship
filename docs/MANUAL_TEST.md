# 手動確認

このファイルは、`index.html` の変更後に既存機能を壊していないか確認するための手順です。  
今回追加した `AI相談JSON出力` と優先度の高い操作改善を中心に、既存の保存 / 読込と出力もあわせて確認します。

## 事前準備

1. `index.html` をブラウザで開く
2. 初期サンプルが表示されることを確認する
3. 左 / 中央 / 右の 3 カラムが崩れていないことを確認する

## 今回追加した機能の確認

### 1. AI相談JSON出力

1. 左カラムに `AI相談JSON出力` ボタンがあることを確認する
2. ボタンを押して `<プロジェクト名>_AI相談.json` がダウンロードされることを確認する
3. 出力JSONに次が入っていることを確認する
   - `format`
   - `version`
   - `projectName`
   - `currentStep`
   - `consultationPurpose`
   - `stepGuide`
   - `graphSummary`
   - `validation`
   - `activeConcerns`
   - `graph`
   - `aiRequest`
4. `validation.step1` / `validation.step2` / `validation.step3` に `ran` `dirty` `issues` が入っていることを確認する
5. `validation.step2` に `strongCount` が入っていることを確認する
6. `graph.nodes` と `graph.edges` が保存用JSONとは別形式の相談用出力として入っていることを確認する
7. この相談JSONが `JSON読込` の対象ではないことを確認する

### 2. ドラッグで箱作成

1. 図エリアの何もない場所をドラッグする
2. ドラッグ中に作成予定の矩形が見えることを確認する
3. マウスを離すと新しい通常箱が作られ、選択状態になることを確認する
4. ごく小さいドラッグでは箱が作られないことを確認する
5. 作成直後に右カラムでカテゴリ / タイトル / 意味合い / 顧客向け注釈 / 要件追加を編集できることを確認する

### 3. 箱リサイズ

1. 箱を選ぶ
2. 右下にリサイズハンドルが表示されることを確認する
3. ハンドルをドラッグして箱の幅と高さが変わることを確認する
4. リサイズ後に線の位置が追従することを確認する
5. JSON保存して読み込み直したあともサイズが再現されることを確認する
6. 箱内要件がある箱でも表示崩れがないことを確認する

### 4. 線をドラッグで接続

1. `線接続を開始` を押す
2. ガイド文が「いまは線をつなぐ状態です」に変わることを確認する
3. 始点にしたい箱から終点にしたい箱へドラッグする
4. ドラッグ中にガイド線が見えることを確認する
5. 始点とつなぎ先が見た目で分かることを確認する
6. マウスを離すと線が作成されることを確認する
7. 箱内要件同士の接続も従来どおり行えることを確認する
8. 重複する同方向の線は作られないことを確認する

### 5. 線接続UIの文言

1. 線接続中に `線接続をやめる` ではなく `箱接続に戻る` が表示されることを確認する
2. 接続ガイドがやさしい日本語で表示されることを確認する
3. 始点選択後に次のつなぎ先案内へ変わることを確認する

### 6. Undo / Redo

1. `直前の操作を戻す` と `戻した操作をやり直す` ボタンがあることを確認する
2. 箱作成後に Undo で元に戻ることを確認する
3. Redo で再び箱が戻ることを確認する
4. 箱移動後に Undo / Redo が効くことを確認する
5. 箱リサイズ後に Undo / Redo が効くことを確認する
6. 線作成後に Undo / Redo が効くことを確認する
7. 箱編集、線編集、箱内要件追加 / 編集 / 削除でも Undo / Redo が効くことを確認する
8. `サンプルに戻す` と `JSON読込` のあとも Undo / Redo が効くことを確認する
9. `Ctrl + Z` で Undo できることを確認する
10. `Ctrl + Y` で Redo できることを確認する

### 7. ズーム

1. 図エリア内で `Ctrl + マウスホイール` を使う
2. 拡大縮小できることを確認する
3. `Ctrl + ↑` と `Ctrl + ↓` でも拡大縮小できることを確認する
4. 画面上に現在の表示倍率が出ることを確認する
5. ズーム後も箱移動、箱選択、線接続、リサイズが大きく崩れず使えることを確認する

### 8. STEP説明の明示

1. STEP1 / STEP2 / STEP3 の説明が左カラムまたは上部で常時見えることを確認する
2. 現在のSTEPに対応する説明が分かりやすく表示されることを確認する
3. STEP切替時に説明表示も切り替わることを確認する

## 既存機能の回帰確認

### 1. 箱 / 親箱 / 箱内要件

1. `箱を追加` ボタンで通常箱を追加できることを確認する
2. `親箱を追加` ボタンで親箱を追加できることを確認する
3. 右カラムで箱のカテゴリ、タイトル、意味合い、顧客向け注釈を編集できることを確認する
4. 箱内要件の追加 / 編集 / 削除ができることを確認する
5. 箱削除で関連線や所属関係が安全側で整理されることを確認する

### 2. 保存 / 読込

1. `JSON保存` で途中保存用JSONを出力できることを確認する
2. 保存したJSONを `JSON読込` で戻せることを確認する
3. 保存 / 読込後も箱位置、箱サイズ、親箱所属、箱内要件、線が再現されることを確認する
4. 読込後に STEP 判定を再実行できることを確認する

### 3. 出力

1. `AI向け出力` がこれまで通り実行できることを確認する
2. `human向け出力` がこれまで通り実行できることを確認する
3. 変更後も出力内容が空にならないことを確認する

### 4. STEP判定

1. STEP1 / STEP2 / STEP3 の判定ボタンが動くことを確認する
2. 判定結果が日本語の文章で表示されることを確認する
3. 図を変更したあとに再判定が必要であることが分かる表示になることを確認する

## 最終確認

1. ブラウザのコンソールにエラーが出ていないことを確認する
2. 今回追加した機能を使っても既存の JSON 保存 / 読込、AI向け出力、human向け出力、STEP判定が壊れていないことを確認する

## 追記: 2026-04-12 初手つまずき改善の確認

### 1. Deleteキー削除

1. 図上で通常箱を選ぶ
2. `Delete` キーを押し、箱が削除されることを確認する
3. 削除に合わせて関連線や所属関係が安全側で整理されることを確認する
4. 図上で線を選び、`Delete` キーで線だけが削除されることを確認する
5. 右カラムの入力欄または textarea にフォーカスを置いたまま `Delete` キーを押し、文字編集だけが行われて図要素は消えないことを確認する
6. 日本語入力の変換中に誤って図要素が消えないことを確認する
7. `Undo` / `Redo` と `Ctrl + Z` / `Ctrl + Y` でも削除前後に戻せることを確認する

### 2. 親箱ドラッグ作成と親箱所属

1. 通常時に空き領域をドラッグし、これまで通り通常箱が作成されることを確認する
2. `親箱を追加` を押し、ガイド文が親箱作成モード向けに変わることを確認する
3. 次の空き領域ドラッグで親箱が作られ、作成後に通常モードへ戻ることを確認する
4. 通常箱を親箱の中へドラッグ移動して離し、右カラムの `属する親箱` と保存データの `parentId` がその親箱になることを確認する
5. 通常箱を親箱の外へ十分に出して離し、`parentId` が空に戻ることを確認する
6. 親箱自身を別の親箱へ入れても子要素化されないことを確認する
7. 親箱が重なる配置では、1つだけに所属し、保存後もその所属先が再現されることを確認する
8. 所属変更後も線表示、箱移動、箱リサイズ、線接続が大きく崩れないことを確認する
9. `JSON保存` → `JSON読込` 後も親箱所属が保持されることを確認する

### 3. AI生成ひな型JSONの読込

1. AIに返してもらう保存用JSON互換データとして、少なくとも `version` `projectName` `currentStep` `nodes` `edges` を持つJSONを用意する
2. `JSON読込` でそのJSONを読み込み、箱 / 親箱 / 箱内要件 / 線 / STEP / 親箱所属が画面に反映されることを確認する
3. 読み込んだ直後に `STEP1` / `STEP2` / `STEP3` を再判定できることを確認する
4. 既存の途中保存JSONも、これまで通り `JSON読込` で安全に戻せることを確認する
5. `AI相談JSON出力` で作ったJSONを `JSON読込` に入れた場合は、読込対象外であることが分かる案内が出ることを確認する
