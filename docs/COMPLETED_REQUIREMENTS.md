# 達成済み要件

## この文書の役割
- この文書には、現時点で達成済みの要件だけを記述します。
- 未達の項目を達成したら、`docs/UNMET_REQUIREMENTS.md` から削除し、この文書へ追記してください。

---

## 達成済みの項目

### 画面構成
- ローカルでブラウザから開ける単一 HTML 構成
- 左 / 中央 / 右の 3 カラム構成
- 日本語 UI
- 開始時に `空の状態から始める` `JSON読込から始める` `サンプルから始める` を選べる
- 左右パネルをツールバーから開閉できる
- `canvas-toolbar` に現在 STEP ですることを表示できる
- `canvas-toolbar` にショートカット説明のアコーディオンを表示できる
- 左カラム下部の使い方を見失いにくい位置で表示できる

### 箱・親箱・箱内要件
- 図エリアのドラッグで通常箱を作れる
- ボタンから通常箱を追加できる
- 親箱作成モードで親箱 / グループ箱を作れる
- 箱をドラッグ移動できる
- 箱をリサイズできる
- 箱のカテゴリ、タイトル、意味合い、顧客向け注釈を編集できる
- 通常箱を親箱に所属させられる
- 箱を親箱内へ動かしたときに所属を自動補正できる
- 箱内要件の追加 / 編集 / 削除ができる

### 線
- 箱同士を線でつなげられる
- 箱内要件同士を線でつなげられる
- 線の関係種別、説明ラベル、`arrowStart`、`arrowEnd` を編集できる
- `Ctrl / Cmd + クリック` で線接続を始められる
- 選択中の箱 / 線は画面上で判別できる

### 保存・読込・出力
- 保存用 JSON をダウンロードできる
- 保存用 JSON を読込んで状態復元できる
- 旧形式の `boxes` / `lines` / `connections` を吸収して読込める
- 不正な親箱参照や存在しない接続先を安全側に補正して読込める
- AI向け Markdown を `_AI.md` で出力できる
- human向け Markdown を `_human.md` で出力できる
- AI相談用 JSON を `_AI相談.json` で出力できる

### STEP と補助機能
- STEP1 / STEP2 / STEP3 を切り替えて確認できる
- 各 STEP の判定を日本語文章で表示できる
- 警告が残った状態で STEP2 / STEP3 へ進むと確認ポップアップを出せる
- STEP3 を完了扱いにする前に確認ポップアップを出せる
- 現在状態の要約を右カラムに表示できる
- Undo / Redo が使える
- ズームが使える
- `Ctrl / Cmd + S / O / Z / Shift + Z / Y / 1 / 2 / 3 / + / - / 0` などのショートカットを使える
