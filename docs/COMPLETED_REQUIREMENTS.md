# 達成済み要件

## この文書の役割
- この文書には、現時点で達成済みの要件だけを記述します。
- 未達の項目を達成したら、`docs/UNMET_REQUIREMENTS.md` から削除し、この文書へ追記してください。
- 現在の実装状態は、`index.html` とこの文書を合わせて確認してください。

---

## 達成済みの項目

### 画面構成
- ローカルでブラウザから開ける単一 HTML 構成
- 左 / 中央 / 右の 3 カラム構成
- 日本語 UI
- 図エリアの作業領域を広げ、より広いキャンバスで箱を置いて整理できる
- アプリ内タイトルを `AI実装のための要件整理ツール` にできる
- 開始時に `空の状態から始める` `JSON読込から始める` `AIとの対話を箱と線で見える化しながら土台を作る` `サンプルから始める` を選べる
- 開始画面の 6 候補を同じボタン形式のサムネイル風カードで表示できる
- 開始画面で上下左右キーを使って候補を切り替えられる
- スタート画面で AI へ貼る文章を表示し、コピペ用ボタンでコピーできる
- スタート画面の AI へ貼る文章に、記述したJSONファイルをダウンロードリンクで出す依頼を含められる
- スタート画面の AI 導線から保存用 JSON をドラッグ＆ドロップまたは選択して読み込める
- 開始画面に `userLevel` の4段階選択UIを表示できる
- 開始画面で `userLevel` を選んでから `空の状態から始める` `サンプルから始める` `AIとの対話を箱と線で見える化しながら土台を作る` を使える
- 左側をスリムなフローティングツールバーとして表示し、開くと元の左パネル相当の内容を見られる
- 右パネル側のボタンで詳細パネルを折りたたみ / 再表示できる
- `canvas-toolbar` に現在 STEP ですることを表示できる
- ショートカットキーの確認を左ツールバーへ置き、閉状態のドックと開状態のアコーディオンから見られる
- `canvas-toolbar` の説明文が極端に細く折り返しにくい配置にできる
- PC 幅で `canvas-toolbar` の状態バッジを 1 行に並べられる
- PC 幅で `STEP` ガイドと接続ガイドを左右に分けて配置できる
- `canvas-toolbar` の `toolbar-side` 内で `connect-guide` の下に現在の `userLevel` を表示し、開始後も変更できる
- `userLevel` の表示文言を短くし、開始画面と `canvas-toolbar` で `はじめて` `少し` `ある程度` `エンジニア` の4段階ラベルを見せられる
- `canvas-toolbar` の `userLevel` 操作を4ボタン1行にし、現在レベルのバッジを `status` 内で見られる
- 開始画面では `userLevel` の詳細文言を表示し、`canvas-toolbar` では短いラベル表示を維持できる
- 開始画面の `userLevel` 注記で、選択中レベルの補足文と保存JSON / AI向け出力への反映案内を併記できる
- 左カラム上部の使い方をアコーディオンで開閉できる
- `使い方` に開閉できることが分かる印を表示できる
- 左ツールバーを閉じているときも、使い方、ショートカット、Undo / Redo、保存、読込、始め方の導線を残せる
- 左ツールバーを閉じているときも `S1 / S2 / S3` で STEP を切り替えられる

### 箱・親箱・箱内要件
- 図エリアのドラッグで通常箱を作れる
- ボタンから通常箱を追加できる
- 親箱作成モードで親箱 / グループ箱を作れる
- 箱をドラッグ移動できる
- 箱をリサイズできる
- `Shift + ドラッグ` で図エリアを上下左右にパンできる
- 箱のカテゴリ、タイトル、意味合い、顧客向け注釈を編集できる
- 箱の見出しをカテゴリごとに色分けして見分けやすくできる
- 通常箱を親箱に所属させられる
- 箱を親箱内へ動かしたときに所属を自動補正できる
- 箱内要件の追加 / 編集 / 削除ができる
- 箱を選択した状態で `Ctrl / Cmd + C` でコピーし、`Ctrl / Cmd + V` で複製できる
- 箱内要件を独立した選択対象として扱え、図上・詳細編集・現在状態の要約で選択中を分かるようにできる
- 箱内要件の入力で 1 文字ごとにフォーカスが外れず、連続入力できる
- 箱を狭くしても、箱内要件を折り返しと内部スクロールで収められる
- `整列` ボタンで箱を見やすく並べ直せる

### 線
- 箱同士を線でつなげられる
- 箱内要件同士を線でつなげられる
- 線の関係種別、説明ラベル、`arrowStart`、`arrowEnd` を編集できる
- 線を関係種別ごとに色分けして見分けやすくできる
- `Ctrl / Cmd + クリック` で線接続を始められる
- 選択中の箱 / 線は画面上で判別できる
- 箱や線を削除する前に確認メッセージを出せる
- 親から子、子から親、子から子の接続でも矢印の向きを見える形で表示できる

### 保存・読込・出力
- 保存用 JSON をダウンロードできる
- 保存用 JSON を読込んで状態復元できる
- 旧形式の `boxes` / `lines` / `connections` を吸収して読込める
- 不正な親箱参照や存在しない接続先を安全側に補正して読込める
- `userLevel` を保存用 JSON に保持し、再読込時に復元できる
- `userLevel` が無い保存JSONを読込んだときは `はじめて` を採用できる
- AI向け Markdown を `_AI.md` で出力できる
- AI向け Markdown に `userLevel` と説明粒度の案内を含められる
- human向け Markdown を `_human.md` で出力できる
- AI相談用 JSON を `_AI相談.json` で出力できる
- AI相談JSON に `userLevel` と説明粒度の案内を含められる

### STEP と補助機能
- STEP1 / STEP2 / STEP3 を切り替えて確認できる
- 各 STEP の判定をサイドバーを開かず、上部ツールバーからも実行できる
- `Ctrl / Cmd + Shift + 1 / 2 / 3` で STEP1 / STEP2 / STEP3 の判定を実行できる
- 各 STEP の判定を日本語文章で表示できる
- 警告が残った状態で STEP2 / STEP3 へ進むと確認ポップアップを出せる
- STEP3 を完了扱いにする前に確認ポップアップを出せる
- 現在状態の要約を右カラムに表示できる
- Undo / Redo が使える
- ズームが使える
- `Ctrl / Cmd + S / O / Z / Shift + Z / Y / 1 / 2 / 3 / + / - / 0` などのショートカットを使える
