# 仕様固定書

## この文書の役割
- 本書は、ユーザー要望として固定する仕様をまとめる文書です。
- 仕様として古くなった記述は残さず、現行の要望に合わせて更新してください。
- `REQUIREMENTS.md` で確認と承認が取れた内容だけを本書へ反映してください。
- 2026-04-17 追記: ユーザー要望により、既存仕様を変更する場合は元記述を残し、その下に日付と変更内容を追記して運用してください。この追記運用を現行ルールとして優先します。

---

## 目的
このツールは、非エンジニアでも要件・関係性・入出力・保存先 / 参照先を箱と線で整理し、STEP1 / STEP2 / STEP3 で段階確認しながら、顧客説明用と AI / エンジニア解釈用の情報へ落とし込めるローカル動作の単一 HTML ツールとする。

このツールは、見た目だけの図ツールではなく、関係性の整理、説明、合意形成、AI相談の土台づくりに使えることを目的とする。

---

## 想定利用者
- 主な利用者は非エンジニア
- 顧客も非 IT 人材を想定する
- AI やエンジニアへ引き渡す前段の整理に使う

---

## 固定するタイトル
- アプリ内タイトルは `AI実装のための要件整理ツール` とする

---

## 固定する画面構成
画面は中央の図エリアを主役にしつつ、左のツールバーと右の詳細パネルを組み合わせる構成を基本とする。

### 左カラム
- 左側は太い固定パネルではなく、細いフローティングツールバーとする
- 普段は幅 50px 前後のスリム棒として表示し、アイコン中心で操作できる
- ハンバーガーメニューを開いたときは、元の左パネル相当の内容が見える通常表示に戻す
- ハンバーガーメニューを閉じたときだけ、細い棒の状態に戻す
- ツールバーから基本設定、箱追加、親箱追加、線接続開始、保存、読込、STEP 切替、判定実行、出力へ触れられる
- STEP の説明一覧は左カラムに置かない
- `使い方` は展開パネル上部でアコーディオン開閉できるようにし、初見で見つけやすくする
- `使い方` の見出しには、開閉できることが分かる記号を付ける

### 中央
- 図エリア
- `canvas-toolbar` に現在 STEP ですることを表示する
- ショートカットキーの確認は左ツールバーに置き、閉じているときはドックから開き、開いているときは `使い方` の下でアコーディオン表示する
- `canvas-toolbar` の説明文は、ブラウザ幅が狭いときも極端に細かく折り返さず読める配置にする
- PC 幅では `canvas-toolbar` の状態バッジを 1 行に並べる
- PC 幅では `STEP` ガイドと接続ガイドを左右に分けて、ツールバー全体の高さを抑える
- 箱配置
- 親箱表示
- 線表示
- 箱内要件表示
- 選択状態の視覚化
- ドラッグ移動
- `Shift + ドラッグ` で図エリア内を上下左右にパンできる

### 右カラム
- 選択中要素の詳細編集
- 箱編集
- 線編集
- 箱内要件編集
- STEP 結果表示
- 現在状態の要約
- 右パネルの開閉ボタンは右パネル側に置く
- 右パネルを閉じたときも、細い状態で再オープンできる導線を残す

## 開始時の仕様
- 開始時は、進め方を選ぶ画面を表示する
- `空の状態から始める`
- `JSON読込から始める`
- `AIとの対話を箱と線で見える化しながら土台を作る`
- `サンプルから始める`
- AI 導線を選ぶと、AIへ貼る文章、コピペ用ボタン、保存用JSONのドラッグ＆ドロップ読込欄を表示する
- 開始画面の 6 つの候補は、すべて同じボタン形式のカードで表示する
- 開始画面の候補は YouTube のサムネイルのような見た目のカードにし、白い再生三角は置かない
- 開始画面では左右上下キーで候補を切り替えられるようにし、選択中の候補は枠線を強めて少し濃い背景で分かるようにする

---

## 固定する箱カテゴリ
以下のカテゴリを固定とする。

- 顧客要望
- 画面表示
- ユーザー入力
- 保存データ
- 参照データ
- 出力結果
- 注釈

カテゴリの追加・削除は、ユーザーが明示した場合のみ行う。

---

## 固定する線の関係種別
以下の関係種別を固定とする。

- 表示
- 入力
- 保存
- 参照
- 出力
- 関連

関係種別の追加・削除は、ユーザーが明示した場合のみ行う。

---

## 扱う要素

### 箱
- 通常の箱を持てる
- 親箱 / グループ箱を持てる
- 箱にはカテゴリ、タイトル、意味合い、顧客向け注釈を持たせる
- 箱は移動、リサイズ、削除できる

### 箱内要件
- 箱の中に箱内要件を持てる
- 箱内要件は `id` `title` `description` を持つ
- 箱内要件は追加 / 編集 / 削除できる
- 箱を狭くしても、箱内要件は折り返しと内部スクロールで箱内に収める

### 線
- 箱同士を線でつなげられる
- 箱内要件同士を線でつなげられる
- 線には関係種別、説明、矢印方向を持たせる
- 線の矢印は、親子関係や接続方向に関わらず見えるように表示する

---

## コア操作
- 箱をボタンまたは図エリアのドラッグで作れる
- 親箱を作れる
- 箱を移動できる
- 箱をリサイズできる
- 箱や線を選択して編集できる
- `選択中を削除` では、削除前に確認メッセージを出す
- 線接続モードでドラッグ接続できる
- `Ctrl / Cmd + クリック` で線接続を始められる
- `整列` ボタンで箱の位置と見え方を整理できる
- Undo / Redo が使える
- ズームが使える
- 左右パネルを折りたためる
- 左はフローティングツールバーとして開閉でき、開いたときは通常の左パネル相当の内容を見られる
- 左ツールバーを閉じているときも、使い方、ショートカット、Undo / Redo、保存、読込、始め方の導線を残す
- 左ツールバーを閉じているときも、`S1 / S2 / S3` で STEP を切り替えられる
- よく使うショートカットキーを使える
- 箱の見出しはカテゴリごとに色分けする
- 線は関係種別ごとに色分けする

---

## STEP の役割

### STEP1
- 顧客要望や大まかな関係性を可視化する
- 主要カテゴリの不足、未入力、独立箱、線なしを確認する

### STEP2
- 保存先 ⇔ 参照先を最優先で確認する
- 保存、参照、表示、入力のつながりの不足を確認する

### STEP3
- 顧客にも作成者にもわかる粒度まで説明を落とす
- 箱、箱内要件、線の説明不足を確認する

---

## 保存 / 読込
- 保存と読込はブラウザだけで完結する
- 保存用 JSON で途中状態を残せる
- 保存対象は `version` `projectName` `currentStep` `nodes` `edges`
- 旧形式の `boxes` / `lines` / `connections` も読める範囲で吸収する
- 不正な参照は安全側に補正する

---

## 出力

### AI向け出力
- 一意 ID を含む
- 箱、親箱、箱内要件、線、関係種別、方向、STEP 状態を含む
- 顧客要望と関連要素の対応が分かる

### human向け出力
- 自然文で読める
- 図全体の目的、各箱の意味、各線の意味、流れ、顧客向け注釈を含む

### AI相談JSON出力
- AI に不足点、確認事項、追加候補を相談しやすい別形式 JSON とする
- 保存 / 再開用 JSON とは分けて扱う

---

## 守る前提
- アプリ本体は `index.html` 1 ファイルにまとめる
- CSS と JavaScript は `index.html` 内に含める
- 外部ライブラリ、フレームワーク、CDN、ビルドツール、バックエンドは禁止
- UI 文言、エラーメッセージ、補助文はやさしい日本語を使う
- 見た目だけの図ツールにせず、判定と説明文出力を重視する
