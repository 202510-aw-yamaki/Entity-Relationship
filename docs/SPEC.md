# 仕様固定書

## この文書の役割
- 本書は、ユーザー要望として固定する仕様をまとめる文書です。
- 仕様として古くなった記述は残さず、現行の要望に合わせて更新してください。

---

## 目的
このツールは、非エンジニアでも要件・関係性・入出力・保存先 / 参照先を箱と線で整理し、STEP1 / STEP2 / STEP3 で段階確認しながら、顧客説明用と AI / エンジニア解釈用の情報へ落とし込めるローカル動作の単一 HTML ツールとする。

このツールは、見た目だけの図ツールではなく、関係性の整理、説明、合意形成、AI相談の土台づくりに使えることを目的とする。

---

## 想定利用者
- 主な利用者は非エンジニア
- 顧客も非 IT 人材を想定する
- AI やエンジニアへ引き渡す前段の整理に使う

---

## 固定する画面構成
画面は 3 カラム構成を基本とする。

### 左カラム
- 基本設定
- 箱追加
- 親箱追加
- 線接続開始
- 保存
- 読込
- STEP 切替
- STEP の説明一覧は左カラムに置かない
- 判定実行
- 出力
- 使い方の短い説明
- 使い方は左カラム下部でアコーディオン開閉できるようにする

### 中央
- 図エリア
- `canvas-toolbar` に現在 STEP ですることを表示する
- `canvas-toolbar` にショートカット説明のアコーディオンを置く
- `canvas-toolbar` に左右パネルの開閉ボタンを置く
- 箱配置
- 親箱表示
- 線表示
- 箱内要件表示
- 選択状態の視覚化
- ドラッグ移動

### 右カラム
- 選択中要素の詳細編集
- 箱編集
- 線編集
- 箱内要件編集
- STEP 結果表示
- 現在状態の要約

---

## 開始時の仕様
- 開始時は、進め方を選ぶ画面を表示する
- `空の状態から始める`
- `JSON読込から始める`
- `サンプルから始める`

---

## 固定する箱カテゴリ
以下のカテゴリを固定とする。

- 顧客要望
- 画面表示
- ユーザー入力
- 保存データ
- 参照データ
- 出力結果
- 注釈

カテゴリの追加・削除は、ユーザーが明示した場合のみ行う。

---

## 固定する線の関係種別
以下の関係種別を固定とする。

- 表示
- 入力
- 保存
- 参照
- 出力
- 関連

関係種別の追加・削除は、ユーザーが明示した場合のみ行う。

---

## 扱う要素

### 箱
- 通常の箱を持てる
- 親箱 / グループ箱を持てる
- 箱にはカテゴリ、タイトル、意味合い、顧客向け注釈を持たせる
- 箱は移動、リサイズ、削除できる

### 箱内要件
- 箱の中に箱内要件を持てる
- 箱内要件は `id` `title` `description` を持つ
- 箱内要件は追加 / 編集 / 削除できる

### 線
- 箱同士を線でつなげられる
- 箱内要件同士を線でつなげられる
- 線には関係種別、説明、矢印方向を持たせる

---

## コア操作
- 箱をボタンまたは図エリアのドラッグで作れる
- 親箱を作れる
- 箱を移動できる
- 箱をリサイズできる
- 箱や線を選択して編集できる
- 線接続モードでドラッグ接続できる
- `Ctrl / Cmd + クリック` で線接続を始められる
- Undo / Redo が使える
- ズームが使える
- 左右パネルを折りたためる
- よく使うショートカットキーを使える

---

## STEP の役割

### STEP1
- 顧客要望や大まかな関係性を可視化する
- 主要カテゴリの不足、未入力、独立箱、線なしを確認する

### STEP2
- 保存先 ⇔ 参照先を最優先で確認する
- 保存、参照、表示、入力のつながりの不足を確認する

### STEP3
- 顧客にも作成者にもわかる粒度まで説明を落とす
- 箱、箱内要件、線の説明不足を確認する

---

## 保存 / 読込
- 保存と読込はブラウザだけで完結する
- 保存用 JSON で途中状態を残せる
- 保存対象は `version` `projectName` `currentStep` `nodes` `edges`
- 旧形式の `boxes` / `lines` / `connections` も読める範囲で吸収する
- 不正な参照は安全側に補正する

---

## 出力

### AI向け出力
- 一意 ID を含む
- 箱、親箱、箱内要件、線、関係種別、方向、STEP 状態を含む
- 顧客要望と関連要素の対応が分かる

### human向け出力
- 自然文で読める
- 図全体の目的、各箱の意味、各線の意味、流れ、顧客向け注釈を含む

### AI相談JSON出力
- AI に不足点、確認事項、追加候補を相談しやすい別形式 JSON とする
- 保存 / 再開用 JSON とは分けて扱う

---

## 守る前提
- アプリ本体は `index.html` 1 ファイルにまとめる
- CSS と JavaScript は `index.html` 内に含める
- 外部ライブラリ、フレームワーク、CDN、ビルドツール、バックエンドは禁止
- UI 文言、エラーメッセージ、補助文はやさしい日本語を使う
- 見た目だけの図ツールにせず、判定と説明文出力を重視する
