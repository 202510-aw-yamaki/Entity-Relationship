# 仕様固定書

## 目的
このツールは、非エンジニアでも要件・関係性・入出力・保存先・参照先を箱と線で整理し、STEP1 / STEP2 / STEP3 で段階確認しながら、顧客説明用と AI / エンジニア解釈用の情報へ落とし込めるローカル動作の単一 HTML ツールとする。

本書は、以降の実装で仕様がぶれないように、現時点の要求を固定するための文書である。

## 固定する画面構成
画面は 3 カラム構成を基本とする。

### 左カラム
- 操作パネル
- 箱追加
- 線接続開始
- 保存
- 読込
- STEP 切替
- 判定実行
- 出力
- 使い方の短い説明

### 中央
- 図エリア
- 箱配置
- 線表示
- 選択状態の視覚化
- ドラッグ移動

### 右カラム
- 選択中要素の詳細編集
- 箱編集
- 線編集
- STEP 結果表示
- 現在状態の要約

## 固定する箱カテゴリ
以下のカテゴリだけを現フェーズで固定する。

- 顧客要望
- 画面表示
- ユーザー入力
- 保存データ
- 参照データ
- 出力結果
- 注釈

カテゴリの追加・削除は、今後ユーザーが明示した場合のみ行う。

## 固定する線の関係種別
以下の関係種別だけを現フェーズで固定する。

- 表示
- 入力
- 保存
- 参照
- 出力
- 関連

関係種別の追加・削除は、今後ユーザーが明示した場合のみ行う。

## 箱データ構造
箱は最低でも以下を持つ。

| 項目 | 必須 | 説明 |
| --- | --- | --- |
| `id` | 必須 | 箱の一意 ID |
| `category` | 必須 | 箱カテゴリ |
| `title` | 必須 | 箱の見出し |
| `meaning` | 必須 | 箱の意味合い説明 |
| `customerNote` | 必須 | 顧客向け注釈 |
| `x` | 必須 | X 座標 |
| `y` | 必須 | Y 座標 |
| `width` | 必須 | 幅 |
| `height` | 必須 | 高さ |

後続フェーズでのみ追加候補とする項目:

- `parentId`
- `items`

## 線データ構造
線は最低でも以下を持つ。

| 項目 | 必須 | 説明 |
| --- | --- | --- |
| `id` | 必須 | 線の一意 ID |
| `fromType` | 必須 | 接続元の種別 |
| `fromId` | 必須 | 接続元 ID |
| `toType` | 必須 | 接続先の種別 |
| `toId` | 必須 | 接続先 ID |
| `relationType` | 必須 | 関係種別 |
| `label` | 必須 | 線の説明 |
| `arrowStart` | 必須 | 始点側の向き |
| `arrowEnd` | 必須 | 終点側の向き |

`fromType` / `toType` は、現時点では箱同士を前提としつつ、後続フェーズの箱内要件接続も区別できる構造を維持する。

## AI 向け出力の構造
AI 向け出力は構造重視とし、必ず一意 ID を含める。

最低限含める内容:

- プロジェクト名
- 現在 STEP
- 箱一覧
- 線一覧
- 関係種別
- 方向
- 顧客要望と実装関連カテゴリの対応

出力形式は `プロジェクト名_AI.md` または `プロジェクト名_AI.txt` を想定する。

## human 向け出力の構造
human 向け出力は自然文重視とし、非 IT 人材が読める表現にする。

最低限含める内容:

- 図全体の目的
- 各箱の意味
- 各線の意味
- 入力 → 表示 → 保存 → 参照 → 出力 の流れ
- 顧客向け注釈

出力形式は `プロジェクト名_human.md` または `プロジェクト名_human.txt` を想定する。

## 後続フェーズ機能
以下は後続フェーズで扱う候補機能であり、今回の実装対象外とする。

### 箱内箇条書き要件
- 箱内に箇条書きの要件を持てるようにする
- 箱データに `items` を追加候補とする

### 箱内要件同士接続
- 箱内要件同士を線でつなげる
- `fromType` / `toType` で箱と箱内要件を区別できるようにする

### 親箱 / グループ箱
- 箱の上にさらに大きな箱を置けるようにする
- 箱データに `parentId` を追加候補とする

## 今回まだ実装しないもの
今回の仕様固定フェーズでは、以下を実装しない。

- `index.html` の作成
- UI 実装
- 箱追加 / 編集 / 削除の実装
- 線接続 / 編集 / 削除の実装
- ドラッグ移動の実装
- STEP 判定ロジックの実装
- JSON 保存 / 読込の実装
- AI 向け / human 向け出力機能の実装
- 後続フェーズ機能の実装

## 実装時に守る前提
- アプリ本体は `index.html` 1 ファイルにまとめる
- CSS と JavaScript は `index.html` 内に含める
- 外部ライブラリ、フレームワーク、CDN、ビルドツール、バックエンドは禁止
- UI 文言、エラーメッセージ、補助文はやさしい日本語を使う
- 見た目だけの図ツールにせず、判定と説明文出力を重視する
