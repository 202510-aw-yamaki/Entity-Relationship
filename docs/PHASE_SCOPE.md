# フェーズ範囲固定書

## 今回の位置づけ
今回は実装フェーズではなく、仕様固定フェーズとする。アプリ本体にはまだ手を入れず、以降の実装でぶれないように現在の要求を文書として固定する。

## 今回固定した範囲
- 画面構成
- 箱カテゴリ定義
- 線の関係種別定義
- 箱データ構造
- 線データ構造
- STEP1 / STEP2 / STEP3 の判定条件
- 不足メッセージ例
- JSON 保存形式
- AI 向け出力の構造
- human 向け出力の構造
- 後続フェーズ機能の位置づけ
- 今回まだ実装しないもの

## 後続フェーズ機能
以下は元要望に含まれるが、今回の実装対象外とする。

### 1. 箱内箇条書き要件
- 箱の中に箇条書きで要件を持てるようにする
- 箱データの将来拡張として `items` を使う想定とする

### 2. 箱内要件同士接続
- 箱内要件同士を線で接続できるようにする
- 線データの `fromType` / `toType` で区別できる前提を残す

### 3. 親箱 / グループ箱
- 箱の上にさらに大きな箱を置き、まとまりを表現できるようにする
- 箱データの将来拡張として `parentId` を使う想定とする

## 今回まだ実装しないもの
- `index.html` 作成
- 画面レイアウトの実装
- 箱の追加 / 編集 / 削除
- 箱の移動
- 線の接続 / 編集 / 削除
- JSON 保存 / 読込
- STEP 判定処理
- 警告付き STEP 遷移
- AI 向け出力生成
- human 向け出力生成
- 後続フェーズ機能の UI とデータ保持

## 次フェーズで実装する最小範囲
次フェーズでは、以下を最小範囲として扱う。

1. `index.html` 単一ファイルの土台作成
2. 3 カラム画面構成の実装
3. 箱の追加 / 選択 / 編集 / 削除
4. 箱のドラッグ移動
5. 線の接続 / 選択 / 編集 / 削除
6. STEP1 / STEP2 / STEP3 の切替表示の土台
7. 基本 JSON 保存 / 読込

上記は最小範囲であり、後続フェーズ機能は含めない。

## 今回固定しないこと
今回は以下を詳細固定しない。

- 箱や線の初期サイズ・初期色などの細かな見た目
- ボタン配置の細かな余白や装飾
- 判定文の完全な文言テンプレート一覧
- AI 向け / human 向け出力の最終フォーマット詳細
- 後続フェーズ機能の具体 UI

## 注意点
- 実装範囲を勝手に広げない
- 後続フェーズ機能を先回りで入れない
- 単一 HTML を守る
- 非エンジニア向けのやさしい日本語を維持する
- 保存 / 読込互換性を壊さない前提で進める
